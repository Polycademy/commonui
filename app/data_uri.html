Data URIs Demo

Note! Data uris are image data that has been through a base 64 filter. Browsers can interpret this data and essentially build an image from text. The overall size of the data transfer increases by 30%, however you decrease the number of HTTP requests, because these data-uris live in a CSS file. These files should be cached long term to actually get a performanace benefit. There's also the disadvantage of maintanence, because if you need to change the image, you need to change the code, but this can be circumvented by using a serverside language to generate CSS files. Other than potential performance gains, data-uris offers the possibility of dynamically creating images on the fly from Javascript, especially by using the canvas element. Oh and this does not work in IE7 or lower. IE8 is limits data uri image data by 32768 Bytes. Use an online converter to keep under this limit.
